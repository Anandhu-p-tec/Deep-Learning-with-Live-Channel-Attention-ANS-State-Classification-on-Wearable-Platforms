ESP32 Serial Output Format

Expected line format (one line per sample):
GSR:<int>,SPO2:<float>,TEMP:<float>,AX:<float>,AY:<float>,AZ:<float>

Example:
GSR:512,SPO2:98.2,TEMP:36.5,AX:0.10,AY:0.02,AZ:9.80

Notes:
- End each sample with a newline (\n).
- Baud rate: 115200.
- Field order should remain fixed as shown above.
- Numeric values may be integer or float (parser accepts both).
- Avoid extra spaces in keys.

Mapping used by dashboard parser:
- GSR  -> skin conductance proxy
- SPO2 -> oxygen saturation
- TEMP -> body/skin temperature
- AX, AY, AZ -> accelerometer axes (combined as accel magnitude feature)
